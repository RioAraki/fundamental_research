Agent 设计方案 · 可批注

事件 → 大宗价格影响 推理 Agent

Slack 里研究员/交易员问"某事件对某品种价格什么影响"，Agent 基于知识体系 + 内部数据(Wind/Mysteel) 做接地推理，给出方向+幅度+时效+置信度+依据。这份是设计思路，供你逐块批注。

1 · 核心理念：复刻"研究员的推理过程"，不是做问答机器人

最容易踩的坑是把它做成"RAG 问答"——检索几段文档拼个回答。但研究员判断事件影响走的是一条结构化推理链。Agent 的价值不在"知道知识"，而在把知识按研究员的方法论用起来。所以设计的核心是把那条推理链显式编码成 Agent 的工作流。

事件 → 命中哪个因子 → 改变供给/需求/成本/情绪 → 改变供需平衡 → 结合估值(贵不贵) → 方向 × 幅度 × 时效 → 再问"是否已被price in" → 给出可交易结论 + 依据

这条链我们在化工知识库的"PX 价格分析框架"页已经搭过——它就是 Agent 的 decision policy。

2 · 四层架构

知识层(researcher 的脑) + 数据工具层(Wind/Mysteel) + 推理工作流层(researcher 的方法论) + 接口/反馈层(Slack + 纠错回流)。

层职责关键设计
知识层 KB每个品种的驱动逻辑、弹性、季节性、历史类比、关联品种结构化"驱动卡"(见 §4)+ 关系图，可检索、可被 researcher 维护
数据·工具层取数(价格/库存/产能/开工/基差/价差)、算平衡表/分位/价差把 Wind/Mysteel 封装成确定性工具(function calling)，不靠模型记数字
推理·工作流层事件→影响的推理链；可拆成多个子 agent分类器→量化器→质疑者→综合器(见 §3/§6)
接口·反馈层Slack 交互；researcher 打分/纠错回流结论必带引用；纠错沉淀回 KB / few-shot

3 · 事件 → 价格影响 推理链（Agent 的核心 SOP）

步做什么用什么产出
① 解析事件这条新闻/问题涉及哪个(些)品种？命中什么因子(检修/投产/事故/政策/进出口/库存/成本/天气/情绪)？是利多还是利空方向的"种类"新闻解析 + KB 因子分类{品种, 因子, 方向类型}
② 定位知识拉该品种的结构：谁受影响、占产能/需求多少、上下游传导、弹性、季节背景、历史类似事件KB 驱动卡 + 关系图影响路径 + 相关弹性/类比
③ 量化规模事件有多大？相对平衡表/库存意味着什么。例：250万吨检修2周 ≈ 损失多少 vs 月需求/库存天数数据工具(产能/开工/库存/进出口) + 计算对供需平衡的边际冲击(带数字)
④ 推方向·幅度·时效平衡变紧/松 → 价格/价差方向；幅度参考弹性/历史类比；时效(即时 vs 滞后几周)KB 弹性/类比 + 估值分位方向 + 幅度区间 + 时间窗
⑤ 查"已price in"这事市场预期到了吗？看远期曲线/基差/月差/最近走势——预期内的利多可能不涨甚至跌数据工具(盘面/基差) + 推理预期差判断(surprise or not)
⑥ 输出结论给方向/置信/时效/关键假设/失效条件/相关合约价差，逐条标依据综合 + 引用结构化、可审计的结论

⑤ 是把"基本面 agent"和"新闻情绪 bot"拉开差距的关键——研究员的真正价值在于判断预期差，不是复述新闻。

4 · 知识表示：把研究员的脑子变成"驱动卡"

让事件→影响接近"可查表"，关键是把每个品种的知识拆成结构化、机器可读、研究员可维护的驱动卡(Driver Card)。每条因子一行：

字段含义例(PX·装置检修)
factor因子名供给-装置检修/意外停车
observable对应哪个数据字段(去哪取)PX开工率、装置产能(Mysteel)
direction该因子↑对本品价格/价差方向停车→供给↓→PX价/PXN↑
magnitude弹性/敏感度(或历史类比)X万吨/月损失 ≈ ? 库存天数
lag/horizon多久体现、持续多久即时定价，持续=停车时长
transmission向上下游怎么传导PX→PTA加工费、聚酯成本
conditions什么情况下逻辑失效/反转快速重启、下游同步减产
analogs历史类似事件 + 当时价格反应(挂历史案例库)
source知识出处/负责研究员研究员署名 + 校核日期

混合表示更稳：结构化表(给检索/计算)+ 散文解释(给 LLM 推理)+ 关系图(品种→因子→上下游)。我们化工库的品种页其实就是这些卡的"人类可读版"，再蒸馏成机器版即可。

5 · 接地与防幻觉（交易台的生死线）

  • • 数字只能来自工具，不能来自模型记忆：所有量化值经 Wind/Mysteel 工具取，缺数据就明说"无数据"，绝不编。
  • • 三类信息分层标注：事实(来自数据) / 推理(来自知识) / 判断(不确定)——让研究员一眼看出哪些能信、哪些要审。
  • • 强制引用：每个结论挂数据字段 + 知识卡来源；不可引用就不可断言。
  • • 显式展示推理链：把 §3 六步的中间产物亮出来，研究员能审计、能纠错(也是知识回流入口)。
  • • 校准的置信度：宁可给区间+置信，不给虚假精确点位(呼应"基本面给概率不给点位")。

6 · 为什么拆成多个子 Agent

单个大 prompt 容易又快又错。拆成职责单一的子 agent，可分别约束、分别评测：

  • • 分类器：只判定 品种/因子/方向类型(便于路由到对的驱动卡)。
  • • 量化器：只调数据工具算"规模 vs 平衡表"，强约束必须带数。
  • • 质疑者(关键)：专门唱反调——"这是否已 price in？有无反向因子？最弱的假设是哪条？"。基本面最大的错是只看利多不看预期差，独立质疑者能挡住。
  • • 综合器：汇总成结构化结论 + 置信。

这正是我们做知识库时用过的"对抗式验证"模式：先生成、再独立证伪、最后综合。

7 · 一个完整推演例子

提问："某 250 万吨 PX 装置意外停车 2 周，对盘面什么影响？"

  1. 解析：品种=PX；因子=供给·意外停车；方向类型=利多供给收紧。
  2. 定位：KB→该装置占全国 PX 产能约 X%；PX≈98% 去 PTA；影响 PXN 与 PX-PTA 链利润。
  3. 量化(调数据)：250万吨/年 × 2周/52 ≈ 9.6 万吨损失；对比当前 PX 月供给/PTA 原料库存天数 → 边际偏紧 Y%。
  4. 方向/幅度/时效：PX 价、PXN 走强；幅度参考历史类似停车(类比库)；时效≈停车 2 周内，重启后回落。
  5. 已price in?：是"意外"停车=未被预期 → 利多更可能兑现；查基差/月差确认未抢跑。
  6. 结论：偏多 PX/PXN，置信中，窗口 2–4 周；失效条件=提前重启或下游 PTA 同步减产；相关：可观察 PX-PTA 价差。每条标注数据/知识来源。

8 · 评测与信任（没这步交易台不会用）

  • • 历史事件回测：建"黄金集"——过去的事件 + 当时实际价格反应，跑 agent 看方向命中率。
  • • 研究员在环：每次结论可打分/纠错，纠错沉淀回 KB 与 few-shot(持续变强)。
  • • 置信度校准：跟踪"它说高置信时到底对了多少"，校准它别过度自信。
  • • 分层评测：分类器/量化器/质疑者各自有指标，便于定位是哪一环错。

9 · MVP 路线（先窄后宽）

Crawl
单品种(如 PX)+ 只读数据查询 + KB 检索 + §3 结构化模板输出，全程人审、不接交易。先把"推理链+引用"跑通。
Walk
加量化器(自动算规模 vs 平衡表)+ 质疑者 + 回测黄金集；扩到一条链(聚酯链)。研究员在环纠错。
Run
多品种(化工+农产品)+ 新闻实时触发预警 + 置信校准 + 与信号/回测系统对接。

10 · 主要风险与坑

  • • 幻觉数字(已用"工具取数+强制引用"防) · 数据陈旧(标时间戳/口径)。
  • • 双重计价：把已 price in 的利多再算一遍(靠质疑者 + 基差校验)。
  • • 相关≠因果 / regime 变化：历史类比要标注适用条件。
  • • 新闻解析噪声：标题党、重复、口径不一 → 解析层要去重/核源。
  • • 研究员知识是隐性的：很多判断"只可意会"，要靠在环纠错持续显性化。

11 · 需要你拍板的几个决策

  • ① 定位：先做"研究助手"(辅助 researcher) 还是"信号生成"(直接给方向)？建议前者起步。
  • ② 数据接入：Wind/Mysteel 是 API 还是落库？有哪些字段可取、更新频率？这决定量化器能做多深。
  • ③ 知识来源：是把 researcher 访谈/文档蒸馏成驱动卡，还是先用我们这套化工库做样板？
  • ④ 第一个品种 + 第一类事件：建议 PX/PTA + "装置检修/投产"这类规模可量化的事件先打样。
  • ⑤ 技术栈：是否基于 Claude Agent SDK(工具/子agent/eval 都现成)？要的话我可以陪你到代码。
