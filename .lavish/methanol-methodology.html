《甲醇基本面研究》方法论 × 我们的根因图谱

兴证期货 2019《产业链基本面量化研究(一):CMTO 产业链初探》,32 页。核心路线:把基本面逻辑拆成供给/需求/库存三类因子 → 用相关性检验 + 阈值遍历回测验证每条逻辑是否长期有效 → 组合成单边/月差/跨品种策略。

一句话总评:这份报告是我们图谱的"定量验证层"教科书。它和我们做的是同一件事的两半——我们把"利润是供给的调节阀"这类机制画成带传导结构的白箱图(但强度是拍脑袋的);它把同样的机制压成单变量→价格的短路径因子(丢了传导结构),然后用回测证明每条机制长期有效并找出触发阈值。它没有图、没有事件层、没有主题层;我们没有验证、没有阈值、没有 point-in-time 纪律。把它的验证方法套在我们的边上,就是既定路线图里"强度校准"和"condition 参与推理"两步的具体做法。

1. 报告方法论拆解(8 个可辨识的做法,附实证数字)

#做法内容与证据
①供需存三分法因子只留三类:供给(开工率/生产利润/船运/进口利润)、需求(MTO 开工率与利润/传统下游开工率)、库存(华东/华南港口库存)。明确放弃"宏观相关因子回归预测价格"的老路,改为从机制出发挑因子、用回测验证机制
②高频代理替换低频真值 ⭐海关进口是月度且滞后 15 天 → 用日度船运数据(覆盖进口 80%+)+ 日度进口利润做代理;并验证代理有效性:进口利润 vs 月度进口量相关系数 0.52(p<0.0001)
③利润 = 价格的顶和底(双向阀) ⭐生产利润<0 → 成本支撑 → 做多(遍历 K=0/-50/-100/-150 均正收益,胜率 59-86%);MTO 利润<-1200 → 下游停车倒逼 → 做空(胜率 100%,"MTO 利润是甲醇价格上涨的达摩克利斯之剑")。利润机制不是线性的,是阈值触发的
④阈值遍历回测验证机制 ⭐⭐对每条机制:遍历触发阈值(K)/回看窗口(X 期)/边界(Y%),看年化/回撤/夏普/胜率是否稳健。这就是"condition 字段怎么从文字变成数字"的答案
⑤因子逐层叠加增强库存单边:年化 11.4%/胜率 63.8% → 船运+进口利润:5.5%/61% → 船运+进口利润+库存:17.15%/夏普 1.96/胜率 67.5%——供给侧信号必须叠加需求侧(库存)确认才质变,单边供给因子不够
⑥信号冲突 = 不开仓 ⭐发现"开工率-库存"信号与"生产成本"信号冲突时,把冲突时段的信号剔除不交易 → 胜率与夏普大幅提升。给了我们"多空冲突不裁决"之后的行动语义:冲突即观望
⑦Point-in-time 数据纪律 ⭐统一日频、未更新沿用上期(防未来信息);建仓价用"数据可得时点之后"的价格(开工率周四收盘可得→用周四收盘价,库存周五→周五收盘价)
⑧宏观干扰的自觉最大回撤(2015.7-8)复盘:船运高+库存降、基本面偏多,但原油暴跌经烯烃拖垮甲醇——"基本面因子模型需关注宏观因子干扰导致暂时性失效"。另设计 MA-PP 跨品种套利(甲醇 vs MTO 开工率相对强弱)来对冲掉宏观 beta

2. 逐项对照:报告 vs 我们的根因图谱

维度甲醇报告(2019 因子路线)我们(根因图谱三层)判定
知识表达因子直连价格(开工率→价格),传导结构是黑箱,逻辑只存在于文字描述里显式传导图(45/41 节点),每步有方向/时滞/强度/条件/证据,可点击解释我们领先
机制验证相关性检验 + 阈值遍历回测,每条机制有胜率/夏普背书强度是先验拍脑袋(强0.9/中0.6/弱0.3),仅用历史案例定性对照报告领先 ⭐核心借鉴
条件/非线性condition 全部数字化(利润<-1200、船运破 14 期均值 5% 界),阈值经遍历验证condition 是文字("利润窗口持续>1周"),尚不参与推理报告领先 ⭐核心借鉴
数据频率意识高频代理原则 + point-in-time 纪律,工程化完整signals 记了数据源和频率,但无代理关系标注、无可得时点纪律报告领先
事件/主题/时间维度无。因子是连续读数,无事件层;2015 宏观失效只能事后归因,无主题层去解释"为什么这段时间基本面不管用"三层本体(变量/主题/事件)+ 回归特性 + 事件日志 + 分时段结论我们领先
冲突处理冲突信号剔除不开仓(实证有效)多空路径对照输出,不裁决,无行动语义互补:采纳其行动版
输出形态三种表达:单边 / 月差正反套 / 跨品种套利(MA-PP)只输出单边方向(利多/利空)报告领先
可维护性/扩展参数写死在回测里,换品种要重做;有过拟合风险(作者自认无杠杆无止损、系列第一篇)KG-as-code,边表 git 化,新品种半天复制我们领先

3. 具体借鉴清单(5 项,可直接排进路线图)

① 给关键边挂「因子规则 + 验证状态」——强度校准的具体方法

报告证明:一条机制边可以压缩成一条可回测的因子规则。给图里的强边(尤其带 condition 的)增加 factor_rule 与 validation 字段,校准就有了标准动作:相关性检验 → 阈值遍历 → 回填胜率。

# 以铜的"高价抑需"边为例(对照报告 3.2 节 MTO 利润做空)
- {from: SHFE铜价格, to: 中国下游开工率, polarity: "-", lag: 1月内, strength: 中,
   condition: "快涨期下游按需采购",
   factor_rule: "候选规则:精废价差 > 历史 P90 且社库连续 2 周累积 → 看空信号",
   validation: {status: 未回测, method: "阈值遍历(参照兴证甲醇报告 §3.2)", corr: null, win_rate: null}}
# 校准后 strength 不再是拍脑袋:胜率>65% 且稳健 → 强;55-65% → 中;不稳健 → 弱或删边

② signals 增加「高频代理」标注

报告的船运/进口利润 ↔ 海关月度进口,正是我们已有的 ITS 船调 ↔ 马来出口、SPPOMA ↔ MPOB 产量、买船数据 ↔ 海关进口的关系——但我们没显式标注谁是谁的代理、代理覆盖率多少。加 proxy_of 字段,agent 取数时自动优先高频代理、用低频真值定期校验。

③ 冲突的行动语义:冲突 = 观望 + 给出"何时能裁决"

保留"多空路径对照"的展示,但在结论层加一行行动建议:多空分接近(比值>0.5)→ "信号冲突,按报告实证经验此时不开仓";并列出裁决信号(哪个数据先出来就能定方向——正是监测面板的用途)。

④ 输出形态扩展:单边之外,给月差与跨品种表达

报告每个因子都测了"单边 + 正反套"。我们的图天然支持:落在库存/到港节奏上的路径 → 月差表达(近月强弱),跨品种价差节点(豆棕价差、MA-PP)上的路径 → 套利表达(顺带对冲宏观 beta,正是报告用 MA-PP 套利"抵消宏观因素"的思路)。推理输出加一个"建议表达方式"字段即可。

⑤ Point-in-time 纪律写进数据接入规范

将来挂真实数据(MPOB/SMM/NOAA)时:每个指标登记"发布时点",事件日志的 time 用信息可得时刻而非所述事实时刻;回验历史行情时禁止未来信息。这条现在立规矩,成本最低。

5. 建议排期(并入既有路线图)

#动作并入哪一步工作量
1schema 加 factor_rule / validation / proxy_of 三个字段(空值即可,先立规范)随下次边表修订1 小时
2冲突行动语义 + 表达方式建议(单边/月差/跨品种)进推理输出推理器小改半天
3Point-in-time 规范写进数据接入文档"挂真实数据"之前1 小时
4选 2-3 条强边做首批回测校准试点(建议:棕榈油"进口利润→买船"、铜"精废价差→替代"、铜"高价→开工率"),方法照抄报告 §3(相关性检验+阈值遍历)= 路线图"历史回验"的具体化数天,需行情数据
5甲醇本身:该报告的产业逻辑(煤制成本支撑/MTO 利润封顶/港口库存/船运进口)可直接转成 methanol.yaml——它几乎给好了骨架边表和阈值新品种扩展时半天(方法论已验证)

注意报告局限,借鉴时留心:2019 年数据(MTO 占比、进口依存度等需更新);参数遍历取最优有过拟合风险(作者自认是系列第一篇、无杠杆无止损);因子直连价格的短路径丢机制——所以是"借它的验证方法",不是"抄它的参数"。
