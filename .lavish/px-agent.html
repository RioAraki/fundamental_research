PX 基本面推理 Agent

把 PX 研究员「看到一条新闻/事件 → 判断对盘面价格的影响」的思维过程，做成一个可溯源、可校准的推理引擎，回答 trader 在 Slack 里随时抛来的问题。

下面是从「它该回答什么」→「四层架构」→「知识怎么抽」→「怎么推理」→「端到端示例」→「落地路线」的完整设计。每一块都可以直接在页面上批注。

🎯 设计立场：先说清楚我们不做什么

❌ 不是价格预测器。不输出"PX下周到8500"这种点位。基本面给的是方向 + 区间 + 概率，点位预测是资金/情绪的活，会让 agent 失去可信度。
✅ 是研究员的推理副驾。把"这事我怎么看"结构化、可溯源地跑一遍：传导链是什么、量级多大、是否已被 price in、历史上类似事件怎么走。
❌ 不凭记忆编数据。任何数字(产能、开工率、加工费分位)都必须来自 Wind/Mysteel 工具调用，否则明说"缺数据"。
✅ 每个结论都挂依据。输出里每一步都能点回到：哪条数据、哪张因子卡、哪个历史案例。研究员能在 30 秒内 review。

1️⃣ 它要回答什么问题（输入 → 输出）

先定义产品边界。trader/研究员在 Slack 抛过来的，本质都是「某事件 → PX 价格怎么走」。

典型输入（真实 Slack 问法）
韩国 S-OIL 那套 PX 装置检修，影响多大？ 原油涨 5%，PX 能跟涨多少？ 聚酯进入检修季，PX 怎么看？ 现在调油逻辑还在不在？ PXN 加工费现在贵不贵？ 新装置投产这波利空 price in 了吗？
结构化输出（固定模板，方便 review 和回测）
事件 韩国 S-OIL #2 PX 装置（约 XX 万吨/年）非计划停车，预计 2 周
方向 短期偏多 PX，利好 PXN 加工费扩张
区间/量级 影响国内进口供给约 X 万吨/月；PXN 加工费弹性约 +XX 美元/吨（区间 XX–XX）
时滞 1–3 周显现（到港周期 + 库存缓冲）
Price-in ⚠️ 盘面今日已涨 X%，部分计入；增量空间取决于检修是否超预期延长
置信度 中（供给量级可量化，但 price-in 程度不确定）
依据链：① Mysteel 韩国 PX 产能表 → ② 国内进口依存度因子卡 → ③ PXN 加工费历史分位（Wind）→ ④ 2023 类似检修案例

👉 固定输出模板是关键设计：方向/区间/时滞/price-in/置信度/依据链——既是给人看的，也是回测打分的字段。

2️⃣ 四层架构（核心）

从下往上：知识(研究员脑子里的东西)→ 数据工具(Wind/Mysteel)→ 推理(多 agent)→ 接口(Slack)。下层是上层的地基。

L4接口 / 反馈层 — 与人协作
Slack 收发 · 固定模板输出 · 依据溯源(每步可点回数据源) · 研究员一键「同意/纠正」→ 回流校准。这层决定 agent 能不能被信任和持续变好。
L3推理工作流层 — 多 Agent
事件分类器 → 量化器 → 怀疑者(对抗) → 综合器。把研究员的推理拆成可检查的步骤；怀疑者专门做 price-in 检查与反向逻辑，防止"看多就只找看多理由"。
L2数据 / 工具层 — 取数与算数
Wind/Mysteel 接口(价格·加工费·开工率·库存·产能·进口) + 计算工具(PXN 加工费、平衡表、历史分位、调油经济性、基差/月差)。agent 只能通过工具拿数，不能凭记忆。
L1知识层 — 研究员的领域知识（最难、最值钱）
PX 定价模型(成本 + PXN 加工费；下游 PTA→聚酯→涤纶) + 驱动因子卡库(每个基本面因子一张结构化卡：传导链、方向、时滞、弹性、数据源、历史案例、price-in 提示)。这就是把研究员隐性知识"抽"出来的地方——见下一节。
💡 你之前搭的 chem-hub PX 页（6 模块基本面框架 + 定价模型 + 供需看板）其实就是 L1 知识层的人类可读原型。这个 agent 项目，本质是把那一页变成机器可消费的因子卡 + 接上数据工具。

3️⃣ 知识抽取的核心：PX 驱动因子卡

研究员的知识是隐性的（"调油逻辑强的时候 PX 会脱离成本"）。要让 agent 用，必须拆成统一 schema 的卡片。这是你和研究员协作产出的主要工件。

卡片 Schema（每张因子卡的字段）
字段含义为什么 agent 需要它
因子名如「调油经济性」「PTA 开工率」分类器据此匹配事件
所属维度供给 / 需求 / 成本 / 库存 / 估值 / 跨品种保证覆盖不漏维度
传导链因子→…→PX 价格 的因果路径推理可解释，不是黑箱
方向↑因子 → ↑/↓ PX给出方向判断
时滞影响多久显现（天/周/月）区分"今天就动"还是"下月才显"
弹性/量级因子变 1 单位 → PX 变多少（含区间）把"利多"变成"利多 XX 元"
数据源 + 取数口径Wind/Mysteel 哪个字段、什么频率L2 工具据此取数
Price-in 提示这类信息盘面通常多久消化、看哪个指标判断已计入怀疑者据此做 price-in 检查
历史案例2–3 个过去类似事件 + 当时盘面反应给量级和概率提供锚点
失效条件什么情况下这个逻辑不成立防止机械套用
实例卡 ①：调油经济性（PX 的"特色因子"）
维度 供给侧（隐性产能转移）
传导链：汽油裂解价差高 → 调油料(MX/重整油)economics 优于产 PX → 美湾/亚洲炼厂把芳烃拉去调油 → PX 有效供给减少 → PX 偏强、脱离石脑油成本
方向：汽油-石脑油价差 ↑ → PX ↑（PXN 扩张）
时滞：数周（跟随汽油旺季，北半球夏季最强）　弹性：定性为主，强信号年份 PXN 可较成本逻辑额外 +XX 美元
数据源：Wind 汽油裂解价差、RBOB、MX-PX 价差；Mysteel 芳烃流向　频率：日
Price-in 提示：调油是季节性强预期，盘面常提前 1–2 月交易；看 PXN 是否已在高分位
历史案例：2023 夏调油逻辑驱动 PXN 大幅走扩 …（待研究员补具体数）
失效条件：汽油旺季结束 / 原油大跌带崩芳烃 / 调油料政策收紧
实例卡 ②：PTA 开工率（最直接的需求因子）
维度 需求侧（下游刚需）
传导链：PTA 装置开工 ↑ → PX 刚性消耗 ↑（0.66 吨 PX/吨 PTA）→ PX 去库 → 偏强
方向：PTA 开工率 ↑ → PX ↑　时滞：近实时～1 周
弹性：可由平衡表算：PTA 开工每 ±5% ≈ PX 需求 ±X 万吨/月
数据源：Mysteel PTA 周度开工率、聚酯负荷；Wind PTA 加工费(判断 PTA 自身有无检修动机)　频率：周
Price-in 提示：计划检修通常已知、已计入；非计划停车才是增量信息
失效条件：PX 自身大量累库时，需求边际改善被库存吸收

👉 目标：和每位研究员一起，把 PX 的 10–20 个核心因子都填成这样的卡。卡库 = agent 的"大脑"，也是可被审计、可版本管理的资产。

4️⃣ 推理工作流：为什么要多 Agent

单个 prompt 让模型"分析一下"会幻觉、会单边、会忘记 price-in。拆成职责单一的 4 个角色，每步可检查、可对抗。

① 事件分类器

把 Slack 自然语言 → 结构化事件：类型(供给冲击/需求/成本/政策/情绪)、主体、量级、时间。匹配到相关因子卡。

② 量化器

调 L2 工具取数 + 按因子卡的弹性算量级：影响多少供给/需求、PXN 怎么动、当前加工费分位。只用工具返回的数。

③ 怀疑者（对抗）

专职唱反调：盘面是否已 price in？有没有反向因子(累库/成本塌)？历史有没有反例？量化器是否过度自信？这是质量护城河。

④ 综合器

融合量化器与怀疑者 → 固定模板输出：方向+区间+时滞+price-in+置信度+依据链。冲突大则降置信度、明说分歧。

Slack 问题
→
分类 + 选卡
→
取数 + 量化
→
对抗质疑
→
综合输出
⚠️ Price-in 检查是 PX agent 最该死磕的一环。期货已经把一致预期打进价格，研究员真正赚的是"预期差"。一个不查 price-in 的 agent 会反复给出"利多→看涨"的废话。怀疑者角色的存在就是为了这个。

5️⃣ 端到端示例：一条 Slack 问题怎么跑完

输入：「韩国 S-OIL PX 装置非计划停车，对 PX 啥影响？」

  1. ①分类
    类型=供给冲击(进口减少)；主体=韩国 S-OIL；时间=非计划、时长待定 → 命中因子卡【进口依存度】【韩国 PX 产能】【非计划检修】
  2. ②量化
    调工具：Mysteel 该装置产能 ≈ XX 万吨/年 → 月度约 X 万吨；查国内 PX 进口依存度(韩日占比)；查当前 PXN 分位(Wind)。算出供给缺口对平衡表的边际影响。
  3. ③对抗
    怀疑者反问：① 盘面今天涨了 X%，是否已 price in？② 检修若仅 1–2 周，量级有限；③ PXN 已在高分位 → 上行空间受限；④ 同期 PTA 是否检修(需求端对冲)？
  4. ④综合
    方向短期偏多、量级有限且部分已计入、时滞1–3 周、置信度中，附依据链。明确标注不确定性来自"检修时长"和"price-in 程度"。

👉 注意输出没有给点位，而是给了一个研究员能立刻判断"对/不对、要不要追问"的结构化结论。

6️⃣ Grounding & 反幻觉（让研究员敢用）

数字必须来自工具
产能/开工/库存/加工费分位一律工具调用返回，prompt 里禁止模型自己报数；取不到就说"缺该数据"。
因子卡是唯一逻辑来源
传导链、弹性、失效条件都引用卡库，不让模型即兴发挥因果关系。卡库由研究员审核。
依据链强制输出
每个结论挂"哪条数据+哪张卡+哪个案例"，无依据的结论不允许出现在输出里。
不确定就降置信度
怀疑者与量化器冲突大 → 低置信度 + 明写分歧，而不是强行给个肯定答案。

7️⃣ 落地路线图（Crawl → Walk → Run）

Crawl · MVP
单品种 · 单场景 · 半自动
  • 只做 PX、只做"供给检修"一类事件
  • 3–5 张核心因子卡，先手工填
  • 数据先半自动(贴数/少量接口)
  • 输出固定模板，研究员人工 review 每条
  • 目标：验证推理结构对不对
Walk
全因子 · 接数据 · 多 agent
  • PX 全因子卡库(10–20 张)
  • Wind/Mysteel 工具接通,自动取数
  • 四角色 pipeline 上线
  • 接 Slack,研究员"同意/纠正"回流
  • 建回测集:历史事件→当时盘面反应打分
Run
复制到链 · 跨品种联动
  • 模板复制到 PTA/EG…(你已有页面 SOP)
  • 跨品种传导(PX↔PTA↔聚酯利润分配)
  • 主动推送:监控数据异动主动播报
  • 持续用反馈校准弹性与 price-in 规则

👉 PX 跑通后,因子卡 schema + 四角色 pipeline + 输出模板 全部可复用到其他品种——这正好对上你"先把 PX 做成满配样板再复制"的思路。
