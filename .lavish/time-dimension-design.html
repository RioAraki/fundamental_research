时间维度 = 根因 / 事件 两层分开建模

v1 的错误(你指出的):把"MPOB库存超预期"这类事件当成了根因去贴时间型标签,违反了根因中性原则。v2 修正:根因是永续存在的中性变量;事件(新闻)是对根因的一次赋值;影响持续性由两层共同决定。

0. 概念修正:根因 vs 事件,一条判据分清

根因(变量)事件(新闻)
是什么永续存在的中性状态量,任何时刻问"它现在的值是多少"都有意义(哪怕值 = 0/中性)对某个根因的一次带时间戳的赋值/观测,有方向、有幅度
例MPOB库存预期差(每月产生一个读数,平时 ≈0)、国储收抛储立场(平时 = 按兵不动)、产区干旱程度(一直有个当前值)"2025.11 月报库存超预期"、"2021.6.16 宣布抛储"、"NOAA 宣布强厄尔尼诺形成"
放哪里图里的节点(机制骨架,永远在)不进图——进事件日志(推理的输入流)
有没有方向没有(中性)有(±)——方向是事件的属性,不是根因的
时间属性回归特性:被冲击后如何回到基线(见 §1)事件窗口:这次事件本身持续多久(见 §2)
判据(一句话):把一句新闻里的时间和方向剥掉,剩下的名词就是根因;带着时间和方向的整句话是事件。"11 月库存超预期"→ 剥掉 → "库存预期差"(根因)。
"没有新闻时没有影响"在这个模型里天然成立:根因读数在基线(0)时,符号传播的输入就是零,整条传导链静默——图存机制(永远在),事件激活机制(偶尔来)。这正是现有推理器的行为:不给冲击就没有传播,只是我们之前没把"冲击"正式化为带时间戳的事件对象。

0.5 三层的正式定义(本体地基)

🧱 变量(根因)Variable

定义:一个可测量的、持续存在的中性状态量,描述影响目标价格的某个机制的当前状态。图谱的节点,知识的原子单位。
一句话:变量是世界的一块仪表盘,图谱是仪表盘之间的接线图。

判定标准
(四条全过)		① 可测量:任何时刻问"现在的值"都有意义(有数据序列,或至少可构造定性档位;值可以=0);② 中性:名字里不许有方向词(上调/短缺/超预期→那是事件);③ 机制独立:有自己的传导机制,不是其他变量的派生改写;④ 实质性:传导占比值得建模
字段id / type / direction(±语义) / note / signals / case / 回归特性(五类) / theme(挂载的主题标签)
生命周期年-十年。出生=新机制跨过实质性阈值(印尼生柴掺混率生于 2008 B2.5);死亡=机制消失(归档不删)。git 历史是它的年谱
反例"库存超预期"含方向且瞬时→事件;"AI数据中心扩建"用主题给变量命名→不纯,应改"数据中心算力基建需求"

⚡ 事件 Event

定义:对某个变量的一次带时间戳的赋值或观测,来源是新闻/数据发布/公告。不进图,进事件日志,是推理器的输入流。
一句话:事件是某一刻有人读了一眼仪表盘、喊出了读数变化。

判定标准① 有时间戳;② 有方向(±,可带幅度);③ 可归属:必须落到受控词表中某个已有变量上——落不到=要么不相关,要么提示图谱缺变量(这是图谱代谢的发现机制:归属失败的事件是新根因的候选来源)
字段time / root_cause / direction / magnitude / window / source
四个亚型冲击型(禁令/事故——改变现实本身)、数据型(月报/统计——揭示现实,价值在预期差)、确认型(趋势年报——增量≈0,早已计价)、反转型(解除/豁免——回收此前的状态型事件)
生命周期小时-月。影响窗口=min(事件持续期, 变量回归期);过期后留在日志里变成历史案例(校准原料)

🔥 主题 Theme(叙事)

定义:一个或多个变量在某段时期的市场注意力状态——市场当前"正在交易什么故事"。既不是变量也不是事件,是挂在变量上的时变标签。
一句话:主题是市场此刻正盯着哪几块仪表盘讲故事。

判定标准① 依附性:必须能指出捆绑了哪些变量(主题不独立存在,是变量的叙事包装,可多对多);② 时段性:有可辨认的活跃期(出生可考,死亡只能事后检测);③ 注意力可观测:热度有代理量——事件密度/价格敏感度/卖方报告提及率
字段name / status(活跃🔥/降温🌡️/休眠💤) / born / bound_variables / note(点燃逻辑+退场条件)
作用机制主题不改变机制(边还是那些边),改变的是计价强度乘数:活跃主题的路径被立刻甚至过度计价;休眠主题的路径被忽视——机制照常运转,只是"没人交易基本面",直到某天突然收敛
生命周期月-年。六种生灭:点燃(单一大事件/连续同向累积)、存续(借尸还魂/反复横跳)、死亡(证伪/计价/自然退烧)。管理=监测不预言
易混点232 关税同时产生了状态型事件(税率真的变了,事实层,由供需定价)和主题(堰塞湖叙事,注意力层,由资金定价)——豁免时事实回滚(−1 事件)与主题退烧同时发生,但是两回事
三层协作 = 推理的完整输入:图(变量+边)给传导结构;事件给激活(方向×幅度×窗口);主题给计价强度乘数。
判定流程:拿到一句话 → 有时间戳+方向?→ 是事件,归属到变量(归属不了→按入图四条判据考虑新变量)→ 说的是"最近市场都在炒 X"?→ 是主题,标到对应变量上。

0.6 两个追问:根因会增减吗?主题到底是什么?

追问一:根因本身会变化吗?——会,图有代谢。真实例子:

变化例子发生了什么
新增(出生)印尼生柴掺混率2008 年印尼推 B2.5 之前,世界上根本没有"掺混率"这个量——变量随政策诞生;到 B20(2016)后传导占比变大,才值得入图
数据中心算力基建需求(铜)2015 年它"存在"但太小(不足消费 1%),不满足实质性判据、不配入图;2023-25 AI capex 爆发,占新增需求显著份额 → 跨过实质性阈值 → 新增节点。根因的"增加"多数是这样:不是世界里凭空出现,而是从"太小不建模"长大到"必须建模"
美国铜进口关税税率=0 存在了几十年、从不驱动价格,不值得入图;2025.2 启动 232 调查后必须入图——沉睡的变量被政策唤醒
减少(死亡/归档)欧盟棕榈油生柴需求曾是棕油需求大项;RED II 后欧盟趋势性退出棕油生柴 → 传导占比萎缩 → 某天从图里归档(现在我们只留了一条弱边 EUDR,就是它萎缩后的残余)
"废六类"进口批文(铜,历史真例)2020 年再生铜原料新国标(GB/T 38471)后,旧的"废六类批文额度"这个变量死亡,被"再生铜原料标准/关税"取代——变量更替

增减都由入图判据(可测量/中性/机制独立/实质性)裁决;归档不物理删除。git diff 里"新增一个节点"这件事本身,就是一条重要的研究记录。

追问二:主题到底是什么?——用一个受控对照实验讲清

同一条新闻,两个年份,唯一的差别是主题状态:

🗓️ 2021 年 10 月:统计局公布竣工面积同比 −20%。当时"地产链坍塌"主题活跃 → 当天铜价应声大跌,全市场都在交易这个数。
🗓️ 2025 年某月:统计局公布竣工面积同比 −20%。同样的事件、同样的根因、同一条机制边,但主题已休眠 → 盘面几乎无反应,当天大家在交易 TC 和美联储。

根因没变、事件没变、传导机制没变——价格反应却差一个量级。造成这个差别的那个东西,就是主题。
价格 = 事实变化 × 市场注意力。根因层管前者,主题层管后者。没有主题层,模型会错误地预测 2025 年那次 −20% 也该跌 2021 年那么多。
一个好记的类比:公司组织
· 根因 = 部门(常设机构,每天都在产出,各管一摊机制)
· 事件 = 某部门今天交的一份报告(有日期有内容)
· 主题 = 本季度的"战略重点"门牌:挂上牌子的部门,报告直达 CEO(市场立刻计价);牌子摘了,部门照常运转、报告照常产出,只是没人看了。
"挂在根因上的标签"就是这块门牌——它不是新部门(不是图里的新节点),它是贴在已有部门门口的、会换的注意力标记。
为什么主题不能当成根因建节点
拿"2026 供给缺口"(铜)来说:它不对应任何单一变量,而是市场把矿端扰动 + TC 归零 + 品位下降三个变量的读数编织成的一个故事。你没法在图里指着某个节点说"这就是供给缺口"——故事的作用是让这三个节点上的任何新事件都被放大计价。主题:变量 = 多对多,它是叙事包装,不是机制原子。
操作性判别根因主题
它变了,现实中的吨数/美元会变吗?会——供需真实变化(竣工少了,铜真的少用了)不会——只是市场看不看它的问题(注意力变化)
在图里吗?是节点不是节点,是写在节点 YAML 里的一个小字段(见下)
怎么测量?数据序列(竣工面积/税率/库存)新闻密度、价格敏感度、研报提及率
谁给它定价?供需——慢而必然(机制迟早收敛)资金——快而善变(可以提前、过度、也可以完全无视)
例地产竣工面积(变量)"地产链坍塌"(2021-22 的叙事)

"标签"在工程上就长这样——节点 YAML 里的一个字段,不新建任何节点:

- id: 地产竣工景气            # ← 根因(节点),永远在图里
  type: 根因·需求
  theme:                      # ← 主题(标签),挂在节点上、随时间改写
    name: 地产链坍塌
    status: 休眠               # 2021-22 曾为"活跃"
    note: "复活条件 = 超预期政策反转"

1. 根因层:五类「回归特性」(v1 五类的正名——贴给变量,不贴给事件)

回归特性回答:这个变量被一次事件推离基线后,以什么方式回去(或不回去)?这是变量的内在性质,与具体哪条新闻无关。

回归特性行为信息寿命根因示例
⚡ 快速重置型按数据发布周期天然归零:下一期是全新的预期差,上一期作废= 发布间隔(月报 1 个月),但价格计价窗口更短(数日)MPOB库存预期差、(铜)交易所库存周度变化
🌊 事件回收型随现实事件结束而回收,结束本身常是一条反向新闻= 事件持续期(洪水退去、罢工谈妥、备货季结束)产区降雨水平、矿端供给扰动率、马来劳工供给、印度节日备货强度
🔒 状态保持型保持到反转事件出现,期间持续有效;但市场逐渐计价(边际递减)不定,以年计也可能(232 关税、B40)印尼出口限制力度、印度进口关税、美联储政策立场、国储收抛储立场
📈 趋势累积型不回归,单向渐变;单条新闻只是对趋势的确认,增量影响小≈永久(决定价格中枢而非日内)树龄老化程度、矿山资本开支与品位、健康/全球食用消费趋势
🔄 季节周期型日历周期函数,可预测,计价提前周期性季节性产量周期

1.5 边界情形:地产竣工、AI数据中心算根因还是新闻?——引出第三层「主题」

完整的时间尺度其实是三层,主题夹在事件和变量之间:

层时间尺度是什么生命周期怎么管
事件小时-周对变量的一次赋值事件窗口,自然过期
主题(叙事)月-年某个变量在某段时期的市场地位——不是变量本身,是挂在变量上的"当前热度状态"无法预言死期,但可监测退场信号(见下)
变量(根因)年-十年机制本身也有生老病死,只是慢;入图/退图 = 图的代谢
地产竣工 = 干净的根因 ✓
变量(竣工面积)有几十年统计序列,任何时刻都有读数——变量永续。只是它的主题状态从"2021-22 交易主线"降到"休眠"。变量在,热度走了,两层各自记账,不冲突。
AI数据中心扩建 = 用主题给根因命名(命名不纯)⚠️
剥掉叙事,底层变量是「数据中心/算力基建需求」——2015 年也存在(小到没人看),AI 退潮后也会存在(可能沉淀成像家电一样的基荷需求)。建议:节点改名为变量纯名,"AI"降级为挂在节点上的主题标签:theme: {name: AI算力扩建, born: 2023, status: 活跃}。主题死了,变量和边还在;下一个主题(比如某天的量子算力)可以挂到同一个变量上。
"很难说主题什么时候消退"——对,所以不预言,只监测。三个可操作的退场信号(任一触发即降档):
① 关注度死亡:事件密度→0(该根因连续 N 个月无新闻命中);
② 计价死亡:事件仍在发生但价格不再响应(event study 敏感度→0,即"利多出尽");
③ 实体死亡:变量的 materiality 跌破阈值(占需求增量的比重不再显著)。
降档动作 = theme 状态改休眠 + 推理打折,节点和边归档不删除——地产若政策反转、热度可复活,知识不应因退烧而丢失。

变量自己也有生命周期(最慢的一层):「印尼生柴掺混率」这个变量在 B 计划出现之前根本不存在,哪天政策废除它也会死。所以图本身是活文档,节点/边的增删是图的代谢,git 历史就是图的成长史。入图判据(什么配当根因):① 可测量(有数据序列或可构造读数);② 对目标价格有实质传导占比;③ 有独立驱动源(不是其他节点的派生)。退图 = 归档到休眠层,不物理删除。

"国储宣布抛储铜铝锌"(2021.6)−1 事件,当日沪铜 -2.66%"保供稳价"——2021 年政策主题,数月后退场国储收抛储立场:平时读数=按兵不动(0)
"统计局:竣工面积同比 -15%"−1 月度数据事件"地产链坍塌"——2021-22 交易主线,现休眠(关注度死亡:同类数据继续发布但无人交易它)地产竣工景气:几十年统计序列,变量健在
"微软上调数据中心 capex 指引"+1 季度财报事件"AI 算力扩建"——2023 诞生、当前最活跃;若 capex 被证伪将退烧数据中心算力基建需求(改名后):2015 年也存在,AI 退潮后也存在,可能沉淀为基荷需求
同一个词能在三层都出现——这是最容易混淆也最能帮你"感受"的地方:
"累库":本周社库 +3 万吨(事件)/ "累库周期"(主题)/ 库存水平(变量);
"减产":某冶炼厂检修公告(事件)/ "冶炼减产潮"(主题)/ 精铜产量(变量);
"干旱":NOAA 本月 ONI 读数(事件)/ "天气市"(主题)/ 产区干旱程度(变量)。
说话时不区分没关系,进图谱时必须落到变量,进日志时必须落到事件,主题只是挂在变量上的热度标签。
层间数量关系:事件:变量 = 多对一(无数条新闻打同一个变量);主题:变量 = 可以多对多——"232 关税"主题捆绑了关税+COMEX库存+跨市价差三个变量,"供给缺口"主题由四五个矿端事件共同喂养。主题是叙事包装,变量是机制原子。
主题的六种生灭模式(全部有真实案例):单一大事件点燃(232)/ 连续同向事件累积(累库六连增)/ 趋势借尸还魂(树龄)/ 证伪死亡(2023 El Niño)/ 兑现·计价死亡(利多出尽)/ 自然退烧(事件密度衰减,地产)。

2. 事件层:事件日志(新增的一等公民)

事件不进图,进事件日志——一个随时间追加的记录流,是推理器的正式输入(也是未来 LLM 解析新闻的输出格式,对应 FinDKG 的带时间戳四元组思想):

# events.yaml(事件日志,append-only;未来由 LLM 从新闻自动生成、人工审核)
- time: "2025-11-10"
  root_cause: MPOB库存预期差        # 必须命中已有根因(受控词表,防幻觉)
  direction: +1                     # 超预期(利空方向的赋值)
  magnitude: 强                     # 超预期幅度(可选:偏离调查中值几个标准差)
  window: 数日                      # 事件窗口;缺省按根因回归特性查表(快速重置型→数日)
  source: "MPOB 11月报 vs 路透调查"

- time: "2022-04-22"
  root_cause: 印尼出口限制力度
  direction: +1                     # 禁令(收紧)
  window: 待反转                    # 状态保持型:持续到反转事件(5.23 出现 direction:-1 的新事件)
  source: "佐科宣布 4.28 起禁止出口"

好处:① 推理输入从"手写 shocks 字典"升级为"查询事件日志:当前有哪些活跃事件?"——叠加多个活跃事件做当下的综合研判成为可能;② 事件日志本身就是历史案例库(回验校准的原料);③ "主题热度"的动态版 = 该根因近期事件密度,自然涌现。

3. 影响持续性 = 两层合成(回答"怎么区分根因和新闻的影响持续性")

某事件经某条路径的影响窗口:

  开始 = 事件时间 + 路径累计 lag        (lag 是边的属性,已有)
  结束 = 开始 + min(事件窗口, 根因回归期)  (两层各出一个,取短板)

三个例子:
① "11月库存超预期"(快速重置型根因 × 脉冲事件)
   → 影响窗口 ≈ 数日。且下月月报发布时,这个读数彻底作废——
     即使价格没反应完,信息寿命也到了。
② "印尼宣布禁令"(状态保持型根因 × 待反转事件)
   → 影响持续到反转事件(5.23 解除 = 新事件 direction:-1),
     期间边际影响递减(逐渐计价);同时反身回路在图上预告了反转的到来。
③ "年度报告确认马来树龄继续老化"(趋势累积型根因 × 确认型事件)
   → 单次事件增量影响 ≈ 0(早已计价),趋势本身持续作用于远期中枢——
     这解释了为什么慢变量"永久影响"但从不驱动日内行情。

推理输出照 v1 方案升级为分时间段结论(0-1月/1-6月/6月+),但每条路径的窗口现在有了严格的来源:边给 lag,事件给窗口,根因给回归期。

5. 主题热度与机制保鲜(v1 方案不变,且更顺了)

  • 热度 = 根因近期事件密度:有了事件日志,热度不再需要单独的采集通道——某根因 90 天内事件条数/加权频率就是热度指数。地产竣工休眠 = 长期无事件;AI 数据中心活跃 = 事件密集。静态标注(theme.status)作为冷启动,事件日志积累后自动接管。
  • 休眠主题:打折 + 标注,不剪掉(机制知识不因没热度而丢失)。
  • 边保质期:review 字段 + CI 超期提醒,不变。
