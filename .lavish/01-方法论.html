商品基本面根因知识图谱 · 方法论总纲

把研究员脑中的供需传导逻辑,变成可推理、可维护、可解释的知识图谱。本页只讲"为什么这么想"(方法论);"怎么用代码实现"见姊妹页《工程落地》。

一句话:不追求"AI 端到端预测价格",而是把研究员的因果框架显性化成一张带方向/时滞/强度的根因图;符号引擎沿图确定性地算出完整传导链,再把整条链交给 LLM 解读——LLM 读入口(解析事件)、解释出口,全程看得见推理过程但不亲自做传导。图谱是资产,推理可解释,知识可协作维护。

0. 方法论全景(一张图)

三层各司其职:根因层(慢,是图谱本体)给传导结构;事件层(快)激活它;主题层(中)调节计价强度。符号引擎沿根因层算出完整推导链,这条链作为 prompt 交给 LLM 解读,输出分时段、可解释、不裁决冲突的结论。

1. 七条核心原则(不可随意推翻)

#原则为什么
1KG-as-code:知识唯一事实源是 git 里的 YAML 文本;图库/页面都是可再生产物可 diff / 回滚 / blame / PR 审批;和研究员写文档同一套协作
2根因中性化:根因是中性状态量(如"印度进口关税"),名字不含方向词;direction 字段说明 +/− 语义同一根因两个方向都会发生(加税/降税),预设方向会漏掉一半行情
3图存机制、新闻定方向:边的极性是传导机制(起点↑→终点如何);事件以 ±1 冲击输入机制稳定(专家维护),方向时变(新闻决定),两者分离
4三层本体:根因 / 主题 / 事件(见下节)把"机制、注意力、观测"三种不同时间尺度的东西分开,才说得清
5冲突不裁决:多空路径并列输出;行动语义=冲突期观望 + 给裁决信号冲突本身是研究信号;硬凑一个方向反而丢信息
6符号引擎算传导、全链喂 LLM 解读:传导由符号引擎确定性执行(不让 LLM 亲自推、避免幻觉路径);但算出的完整推导链(每步方向/时滞/强度/条件)作为 prompt 的一部分交给 LLM,LLM 读入口(解析事件)、解释出口时全程可见并可引用整条推理链LLM 会把相关当因果、幻觉路径,所以不让它做传导;但它看到完整链条才能给出扎实、可追溯的解读,而非只凭首尾臆测
7人可手动编辑,AI 编辑须人审:一切修改都是 PR;人类可直接手动编辑并提交,AI(读研报/新闻)起草的任何修改必须经人审核才能合入——PR review 就是审批闸门不必发明审批;git 已有 diff/回滚/历史/CI 守门;AI 只起草、人拍板,质量与责任清晰

2. 三层本体:根因 / 主题 / 事件

判据一句话:把一句新闻里的时间和方向剥掉,剩下的名词才是根因。

层定义时间尺度存哪例
🧱 根因可测量、持续存在的中性状态量;图谱节点(含中间传导节点)年-十年YAML 图谱矿端供给扰动率、印度进口关税、地产竣工景气
⚡ 事件对某根因的一次带时间戳 ±赋值(来自新闻/数据)小时-周YAML 事件日志"2025.9 Grasberg 事故"=矿端扰动 +1
🔥 主题一个/多个根因在某段时期的市场注意力状态;挂在节点上的标签月-年节点上的字段"AI 算力扩建""地产链坍塌(休眠)"
根因的回归特性(5 类):被冲击后如何回到基线——快速重置(月报预期差)/ 事件回收(罢工)/ 状态保持(禁令,待反转)/ 趋势累积(树龄,不回归)/ 季节周期。决定事件影响窗口的终点。
主题为什么单独一层:同一条"竣工-20%",2021 主题活跃时大跌、2025 休眠时无反应——机制没变,变的是市场看不看它。主题=计价强度乘数,休眠打折不删除。

易混例子(最能帮你分清三层)

日常说话不用分,但进系统时必须落对层。下面这些都是容易混的:

你会听到这句它其实是为什么容易混 / 正确拆解
"库存又增加了"⚡ 事件"库存"本身是根因(节点);"这次增加了 X 万吨"才是事件;"累库周期"是主题——同一个词三层都出现
"美联储加息了"⚡ 事件根因是"美联储政策立场"(任何时刻都有个当前立场);"这次加息"只是对它的一次 +1 赋值
"Grasberg 出事故了"⚡ 事件听着像个"原因",但它是一次带时间戳的观测;根因是"矿端供给扰动率"(永续状态量,任何时刻都有当前值)
"印尼禁止出口了"⚡ 事件(+1)根因是"印尼出口限制力度"(中性,可加严可放松);禁令是让它 +1 的事件,后来的 flush out 是让它 −1 的另一条事件
"AI 数据中心需求"🔥 主题(不是根因)名字像根因,其实把叙事塞进了命名。根因应是中性的"数据中心算力基建需求"(2015 年就存在、AI 退潮后也在);"AI 算力"是挂在它上面的主题标签,会来会走
"现在是供给缺口行情"🔥 主题像个变量,但它不对应任何单一节点——是把"矿端扰动+TC 归零+品位下降"几个根因编织成的一个故事;主题:根因=多对多
"电网投资很强"🧱 根因(读数)这是根因"中国电网投资"的当前水平(慢变量,月度有读数);"国网公布年度计划+18%"才是一条事件

一句话记法:能问"它现在的值是多少"的 → 根因;带时间戳和方向的一句话 → 事件;"最近市场都在炒 X" → 主题。

3. 推理:QPN 符号传播

在带 ± 号的因果图上,从事件节点出发沿路径传播:符号相乘(串联)、强度相乘(弱边衰减)、时滞累加;环路用"路径不重访 + 回源点记为反馈环"处理。算出的每条路径就是一条可交给 LLM 引用的推导链。

冲突消解:同一节点多空同到 → 不裁决,输出利多路径 vs 利空路径 + 各自强度;冲突=观望等裁决信号。
时滞→分时段:按累计时滞分 0-1月/1-6月/6月+ 三桶,各自汇总——回答"近端偏空但远端偏多"这类结构。
可解释:每条结论=一条完整传导链(带方向/时滞/强度/条件),点节点看利好利空判定,点边看"为什么这个强度"。

每条边只有两种:同向(+)=起点涨则终点涨;反向(−)=起点涨则终点跌。沿链走时盯住每个节点"实际是涨还是跌":

起点  干旱 ↑
  │ 反向(−):干旱与单产反向 → 干旱涨,单产
  ▼    单产 ↓
  │ 同向(+):单产与产量同向 → 单产跌,产量也
  ▼    产量 ↓
  │ 同向(+):产量与库存同向 → 产量跌,库存也
  ▼    库存 ↓
  │ 反向(−):库存与价格反向 → 库存跌,价格
  ▼    价格 ↑        ⇒ 干旱↑ 最终导致 价格↑ = 利多

为什么"有 + 有 − 最后是 +"?——不是把符号相加,而是"负负得正"式地相乘:一条链上偶数个反向(−)边就净同向,奇数个就净反向。这条链有 2 个反向边(干旱→单产、库存→价格),两次"反向"互相抵消 → 净同向,所以"干旱涨"一路走成"价格涨"。(−)×(+)×(+)×(−)=+ 只是这个"数减号个数"的简写记法。

4. 根因定性、事件定量(定量很难,别假装自动)

定性(图谱)相对稳定、可沉淀:根因动了往哪传、经过哪些中间节点、什么方向——这是领域知识,专家维护。难的是定量——"这次影响多大"。要诚实分两类看,难度天差地别:

情况怎么定量覆盖面 / 难点
少数"有数字+有市场预期"的定期数据(MPOB 库存、社会库存、TC…)可算 z-score = (实际 − 市场预期)÷ 历史意外的标准差,幅度是客观数字只覆盖一小部分;而且要求你能拿到"市场预期"这个数(路透/彭博调查,或用历史均值近似)——不总是有
绝大多数事件(政策/突发/罢工/研报观点)没有干净的数值意外,幅度只能靠研究员手动判断 强/中/弱主观、经验驱动,是整套方法里最依赖人、最难的一环;我们不假装它能自动化
  • 如实说:z-score 是"能算就算"的锦上添花(覆盖少数定期数据),不是普遍能力;事件幅度的定量现阶段主要靠研究员手动输入,这是已知的硬骨头,后续只能靠历史行情回测慢慢校准,而非一步到位。
  • 每个根因带的 监测信号(新闻关键词 + 数据源)是帮研究员"该看什么、去哪查"的清单,降低手动判断的成本,但判断本身仍是人来做。
  • 图谱不会因为"给不出精确数字"就失效——它先把方向与结构(定性)说清,幅度留白由人补;这本身已经比"拍脑袋想因果链"进了一大步。

5. 图谱如何帮人发现盲点(核心价值)

人的注意力有限、记忆会淡、容易只看一边。图谱不会。下面三个机制具体怎么运作,各附例子:

① 休眠区巡检 — 治"机制我知道,但我没在看"(注意力盲区,最值钱)

怎么运作:定时任务(如每周)扫描所有 status=休眠 的主题对应的根因 → 取该根因监测数据的当前读数 → 算它在历史分布里的分位 → 若读数已到极端分位(如 >P90 / <P10)但该根因近期几乎无事件(新闻密度≈0) → 判定"基本面在积累势能、而市场和你都没在看" → 置顶进巡检报告。

例:「地产竣工景气」主题 2023 年后休眠。2026 某月竣工数据连续回升到 5 年区间上沿(P92),但盘面无人交易地产逻辑、相关新闻近乎为零。巡检把它挑出:"⚠️ 地产竣工读数已达 5 年高位却处休眠;机制:竣工↑→终端铜需求↑,可能被市场低估,建议复核是否重新激活该主题。" —— 图谱替你记着你两年前放下、如今可能重要的东西。

② 反方路径 — 治"我只想到了一边"(路径盲区 / 魔鬼代言人)

怎么运作:研究员给一个观点(如"看多沪铜")→ 系统对目标价跑传导,挑出所有指向与你观点相反方向的路径(当前活跃事件驱动的优先,其次结构性存在的)→ 每条附:传导链、强度、时滞、前提条件、验证信号。潜台词:这些是你的观点若要成立、必须逐一压过的东西。

例:你写"2026 供给缺口,看多铜"。系统自动列反方:
· [空 0.6] 硫酸价格↑→冶炼利润↑→减产不兑现→精铜供给不减(前提:副产收入持续高;验证信号:SMM 精炼铜月产量);
· [空 0.66] 美国关税豁免→COMEX 溢价崩→全球库存回流→LME 承压(验证信号:COMEX-LME 价差)。
→ 逼你在下多头结论前回答:"为什么这两条压不过缺口逻辑?" 观点从此是可检验的,而非一厢情愿。

③ 边保质期 + 主题计价死亡检测 — 治"我以为它还有效"(时效盲区)

怎么运作(两个子机制):
· 边保质期:每条边带 review.last(最后人工确认时间);月度扫描超期(如 >12 月未审)的边 → 提醒研究员复核方向/强度是否还成立。
· 主题计价死亡:对活跃主题,若它的事件仍在发生(新闻密度不低)但价格对这些事件的敏感度趋于 0(event study:同向事件出现日价格不再反应)→ 判定"利多/利空已出尽、充分计价" → 建议该主题降档为休眠。

例(边保质期):"地产竣工→终端铜需求"这条边 2010s 该配"强",随地产占铜需求比重下降现已配"中";若其 review.last 停在 2024、到 2026 超 12 月未复核 → 巡检提醒"此边 2 年未审,当前地产占比可能又变,请确认强度"。
例(计价死亡):2025 年"232 关税"主题初期每条关税新闻都让 COMEX 剧烈波动;到后期关税消息继续出、但价格几乎不再反应 → 敏感度衰减到近 0 → 系统建议把该主题降档(市场已消化),别再当活跃题材交易。
