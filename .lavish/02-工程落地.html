工程落地 + 公司环境落地准备

方法论怎么变成能跑的代码,以及这个 draft repo 带进公司、结合真实研报后如何尽快发挥作用。方法论本身见姊妹页《方法论总纲》。

现状:一个端到端可跑的 draft——图谱本体(沪铜 41 节点/55 边)+ 推理引擎 + 事件系统 + 运维 CLI + CI + web 服务(FastAPI+React Flow),已在 GitHub、CI 绿、本地能起。缺的是"真实数据/研报的接入"——那正是进公司要做的。

0. 工程全景(灰色=待接入公司环境)

1. 仓库结构

2. 三段式:引擎 / 消费 / 守门

层是什么要点
引擎(kg-mvp)纯 Python、品种无关:build_graph + propagate + events_lib新品种只加一份 YAML,代码零改
消费:CLI运维节律(日 ingest/晨报,周 主题,月 保质期,收盘 异常检测)无人值守可跑;也是 agent 的接口
消费:web 服务FastAPI 4 端点 + React Flow 前端交互图谱 + 情景推演;本地两条命令起
守门:CIschema + golden tests + 事件校验 + 跨品种一致性每个 PR 自动跑,防止改坏图

3. 🏢 公司环境落地准备(本页重点)

目标:draft 进公司、结合真实研报,尽快发挥作用。分"带去前先做"和"进去后的 Day-1/Week-1"。

A. 带去公司前,现在就该补的(让它"即插即用")

事项做什么为什么
数据隔离加 reports/ 意图目录并 gitignore;公司研报/真实数据绝不进这个(公开)repo;或进公司先转私有仓研报有版权/机密,代码与数据必须分离
LLM 接入点就绪加 .env.example(LLM_API_KEY/BASE_URL/MODEL);把 llm_parse() 桩写成"读环境变量→调用"的骨架,到公司填 key 即通研报→事件/边的抽取是最大价值杠杆,接口先备好
数据源适配层把 feeds 读取抽象成 fetch_indicators() 接口,现在读 CSV,到公司实现读 Wind/SMM换数据源不动流水线
Onboarding 文档一页 docs/ONBOARDING.md:同事怎么装、起服务、加事件非技术同事第一天能自己跑起来
PR 机制抽象create_pr() 函数化(本机 gh / 公司 GitLab / 内网 git 可换实现)公司未必用 GitHub,接口留活口

B. 进公司后的落地顺序(让它"尽快有用")

时点动作产出
Day 1装环境起服务,用现有沪铜图谱给研究员/交易员演示,建立"图谱能干嘛"的直觉,收集他们的第一反应共识 + 首批修改意见
Day 1-2接一个真实 LLM,拿 5-10 篇真实研报跑"研报→候选事件/候选边→待审队列",看抽取质量验证核心杠杆是否成立
Week 1研究员审沪铜 55 条边(用公司真实认知校方向/强度/时滞)+ 补公司最关注的根因;研报批量对照补缺失边第一张"公司版"可信图谱
Week 1-2接 1-2 个真实数据源(替换 feeds 桩),数据型事件自动化;上 UI 编辑器让非技术同事录事件图谱开始"活"起来(每日更新)
持续回测校准边强度(甲醇报告方法);扩到更多品种(每个只加一份 YAML)从 draft 到生产

C. 研报是公司最大资产 —— 怎么用它

  • 受控词表是锚:LLM 抽研报时只能把内容映射到已有根因(防幻觉);映射不到的进"未归属队列"= 新根因候选(研报在告诉你图缺了什么)。
  • 研报 diff 补边:agent 读研报,对比"研报讲的机制 vs 图上已有的边",缺的自动生成候选边 PR(带研报引用)——比读新闻高效,研报本就是结构化观点。
  • 历史研报双用:① 冷启动扩图的种子(把研报里的因果框架转成边);② 回测校准的语料(研报观点 vs 后续行情)。
  • 始终人审入图:LLM 只起草候选,进待审队列/PR,研究员点头才进图——质量闸门不能省。

5. 怎么跑(速查)

# 环境
pip install networkx pyyaml fastapi "uvicorn[standard]"   # 后端
cd service/frontend && npm install                          # 前端(首次)

# web 服务(单进程,推荐)
cd service/frontend && npm run build
cd ../backend && python -m uvicorn app:app --port 8000      # 打开 http://localhost:8000

# 运维 CLI(kg-mvp/)
python cli.py ci                    # 质量门(改边表后必跑)
python cli.py morning --today ...   # 晨报(分时段结论)
python cli.py advocate --stance 多  # 反方路径
python cli.py ingest-news --today ...  # 新闻→待审队列

详见 kg-mvp/docs/RUNBOOK.md 与 service/README.md。
